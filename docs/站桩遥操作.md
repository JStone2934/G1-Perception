# 准备

### 查看IP

服务器，G1，PICO同局域网，且服务器和G1还需网线连接

- 服务器局域网IP: 192.168.8.169

- 服务器有线网IP: 192.168.123.xxx (不重要)

- G1局域网IP: 192.168.8.178

- G1有线网IP: 192.168.123.164

  

### 环境配置

根据[G1遥操官方文档](https://github.com/unitreerobotics/xr_teleoperate/blob/main/README_zh-CN.md)

以及[isaac仿真环境配置官方文档](https://github.com/unitreerobotics/unitree_sim_isaaclab/blob/main/README_zh-CN.md)

注: 

	1. vuer报错的话：pip install params_proto==2.13.2
	1. 可能要额外pip install aiortc,uvc

# 仿真遥操

### 启动仿真环境

```shell
# 终端1
cd ~/unitree_sim_isaaclab
conda activate unitree_sim_env
python sim_main.py --device cpu  --enable_cameras  --task  Isaac-PickPlace-Cylinder-G129-Dex3-Joint --enable_dex3_dds --robot_type g129
```



### 启动遥操环境

```shell
# 终端2
cd ~/code_file/xr_teleoperate/teleop
conda activate tv
python teleop_hand_and_arm.py --ee=dex3 --sim --record --img-server-ip 192.168.8.169 --input-mode controller
```

### PICO遥操

进入PICO自带浏览器，输入 https://192.168.8.169:60001 并点击'start'看是否能看到第一视角，然后新建标签页再输入 https://192.168.8.169:8012/?ws=wss://192.168.8.169:8012 进入虚拟空间，点击‘Virtual Ready'进入，完成房间标定以后进入遥操

在遥操环境所在终端可以输入：

- r: 开始遥操
- s: 开始录制
- s: 停止录制
- q: 优雅退出

# 真机遥操

### G1开启相机服务

```shell
# ssh jchen572@192.168.8.178
sudo systemctl stop master_service.service #这玩意占用了head camera
sudo lsof /dev/video4 #如果有输出的话需要sudo kill -9 xxx对应id
cd teleimager/
conda activate teleimager
python -m teleimager.image_server #如果'卡住'的话可能是在等你输入sudo的密码，你可以往上翻一下终端
```

##### (可选)查看页面能否正常显示

```shell
# ssh jchen572@192.168.8.169
cd ~/xr_teleoperate/teleop/teleimager/src
conda activate tv
python -m teleimager.image_client --host 192.168.8.178 #一会儿可以一会儿不行可能是线松了
```



### 启动遥操环境

```shell
# ssh jchen572@192.168.8.169
sudo rm /tmp/cdds.LOG
cd ~/code_file/xr_teleoperate/teleop
conda activate tv
python teleop_hand_and_arm.py --ee=dex3_true --record --img-server-ip 192.168.8.178 --input-mode controller --network-interface enx6c1ff76a8b77 --task-dir ./real_data --task-name test
```

(慎用) 如果连了网线ping不了192.168.123.164，然后运行teleop_hand_and_arm.py秒报错的话

```shell
sudo ifconfig enx6c1ff76a8b77 192.168.123.100 netmask 255.255.255.0 up
ping 192.168.123.164 #不行多试几次
```
